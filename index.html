Recipe Wrangler

Recipe Wrangler is a mobile application centered around recipe management. It was developed in a project based course, Software Engineering, in a 10 week period with a 10 person team. My role in the team was developing the REST API that was used to link the front and back end.

A user is able to add recipes to the database, which can be seen by other users. After which other users can rate your recipe to appreciate its deliciousness

When cooking the desired Recipe a notification will show the chef in training what they need before they begin cooking.

each step can be accessed to ensure that all the proper steps are taken when cooking your new masterpiece.

We split into three main teams for the project: Front-end, Back-end, and Algorithm teams. I was required to be in constant communication so that the REST API was successful and the correct information was displayed to the end user. Together we created different artifacts each week to plan out our development. The source code of the project can be found on our github. Below is a video demonstration of the project and a common student:

  • Team: Erick Soto, Julian Pierce, David Orozco, Sandy Li, Dustin Pilkington, Jessica Lacovelli, Luis Calona, Xiaobin Lin
  • Date: March 2018 - June 2018
  • Category: Mobile Development
