Recipe Wrangler

Recipe Wrangler is a mobile application centered around recipe management. It was developed in a Software Engineering project based course, which spanned a 10 week period which consisted of a 10 person team. My role in the team was developing the REST API that was used to link the front and back end.

A user is able to add recipes to the database, which can be seen by other users. After which other users can rate your recipe to appreciate its deliciousness

When cooking the desired Recipe a notification will show the chef in training what they need before they begin cooking.

each step can be accessed to ensure that all the proper steps are taken when cooking your new masterpiece.

We split into three main teams for the project: Front-end, Back-end, and Algorithm teams. I was required to be in constant communication so that the REST API was successful and the correct information was displayed to the end user. Together we created different artifacts each week to plan out our development. The source code of the project can be found on our github.

I-Team Web Components

I-Team's goal within the span of 10 weeks was to develop a set of web components that would be contributed to a UCSD Library. My team was one of the 13 different ten person sized teams in an Advanced Software Engineering project based course. My role in the team was setting up and maintaining the testing frameworks for the various web components. The testing was Comprehensive and included UI, Browser, and Integration Testing. I-teams website can be found here

I-Team was developed to ensure easy integration by either allowing users to install by using npm or by downloading from github. Our team allowed easy to follow documents so that anyone would be able to join our team which could be found here

Above both the Button and Slider component are shown in action.

The Cascader web component is able to display drop down menus.

The link web component is shown above with the radio component shown below.

The project's main focus was the set up and maintnence of the continous integration pipeline. I spent 5 weeks researching, trying out, and setting up the various testing frameworks. After trying various frameworks such as Selenium, open-wc, Showroom, and TestCafe I found that what was best for the team and future users was WebdriverIO for end to end tests and using Mocha and Chai as assertion checking. The teams github shows all the various contributions done by the team throughout the 10 week class.

Craigslist Lite App

I have developed a product app using Google's new Flutter Framework which allows users to program a mobile app which can run on both IOS and Android. With this Craigslist lite app users are able to add their used or new items to sell them to other users. As a user one is able to add images, contact info, location of item, and a favorite system which allows one to keep track of their favorite items.

A user is able to register and view all the items that are saved onto the database by other users as seen in the photo above and below.
